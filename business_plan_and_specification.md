# 🚀 학부모 익명 소통 및 교육 정보 플랫폼 [Barnet] 사업계획서 및 기능명세서 (v3)

구글 SSO 기반의 '인증된 익명성'을 토대로, 학부모들이 바이럴 광고와 개인정보 유출 걱정 없이 자녀의 학교 생활, 학사일정 및 진짜 교육 정보를 공유할 수 있는 하이브리드 모바일 서비스 **Barnet(반넷)**의 통합 기획 문서입니다.

---

##  PART 1. 사업계획서 (Business Plan)

### 1. 사업 배경 및 해결하고자 하는 문제 (Pain Point)
* **맘카페의 신뢰도 저하**: 기존 포털 카페는 브로커에 의한 허위 리뷰(바이럴 학원 홍보) 및 부동산 선동 등의 정보 왜곡이 매우 심각합니다.
* **익명성과 신뢰의 트레이드오프**: 완전 익명 커뮤니티(예: 블라인드, 에브리타임)는 개인 식별이 안 되어 욕설/비방이 난무하고, 실명제 맘카페는 자녀의 신상 노출 우려로 인해 깊이 있는 학부모 상담이나 학원 불만 리뷰 작성을 꺼립니다.
* **학사 정보의 파편화**: 학교 홈페이지의 공지사항과 가정통신문, 사설 학원 일정이 흩어져 있어 학부모들이 일일이 찾아보는 피로도가 높습니다.
* **학부모 커뮤니티의 역기능과 사회적 우려**: 교사에 대한 익명 저격 및 공동 악성 민원 모의로 인한 교권 침해 문제, 사실적시에 의한 명예훼손 등 법적 리스크, 그리고 동네/학군별 서열화에 따른 조장 행위 등 학부모 커뮤니티가 직면한 고유의 리스크가 존재합니다.

### 2. 핵심 솔루션 및 차별화 전략
* **인증된 가명성(Verified Pseudonym)**: 구글 SSO로 회원 식별은 가능케 하되, 활동명은 시스템이 **[지역명 + 학교/학년 + 랜덤 형용사/명사]** 조합으로만 자동 강제 부여하여 자녀 신상 노출 걱정을 차단합니다.
* **실물 영수증 OCR 인증제**: 학부모가 직접 결제한 학원 영수증이나 학원 앱 수강증을 AI OCR로 자동 판독·검증하여, 광고 업자의 침투를 원천 봉쇄한 100% 신뢰할 수 있는 "진짜 수강 평점" 서비스를 제공합니다.
* **NEIS 학사일정 연동형 대화방**: 공공데이터(교육부 학사 정보)를 연동하여 캘린더 일정을 자동 렌더링하고, 각 일정마다 학년별 전용 소통창을 실시간 개설하여 준비물, 수행평가 대비 팁 등을 교환합니다.

### 3. 수익 모델 (Business Model)
1. **타겟팅 지역 교육 광고 (B2C/B2B)**: 학부모가 등록한 지역(예: 반포동, 대치동) 및 자녀 학년 정보에 기반해 인근 보습 학원, 셔틀버스 서비스, 교육 자재의 맞춤형 광고 지면 송출.
2. **선배 맘 노하우 PDF 마켓**: 활동 지수가 검증된 우수 회원이 작성한 특정 학교의 내신 기출 경향 분석, 과목별 공부법 PDF 교재를 앱 내 포인트 또는 소액 결제로 구매하는 플랫폼 수수료 모델.
3. **B2B 교육 트렌드 데이터 판매**: 지역별 학원 수납 영수증 메타데이터 통계를 익명화 가공하여 대형 학원 체인이나 입시 연구소에 지역별 사교육 트렌드 리포트(평균 학원비, 인기 학원 분포 등) 판매.

### 4. 마케팅 및 확장 로드맵
* **Phase 1 (초기 선점)**: 교육열이 가장 높은 서울 강남 3구(대치, 반포, 개포) 및 부산 센텀 지역을 타겟팅하여 거점 학부모 초기 활성화 유치.
* **Phase 2 (리워드 락인)**: 영수증 인증 성공 시 실질적으로 활용 가능한 5,000P 지급 및 포인트 샵을 통한 기프티콘 교환 서비스로 입소문 확산.
* **Phase 3 (전국화)**: 지역 소모임 카풀, 학교 알림장 자동 번역 등 부가 유틸리티 기능을 확장하여 전국 초/중/고 학부모 필수 앱으로 자리매김.

### 5. 리스크 관리 및 사회적 책임 (Risk Management & Compliance)
* **교권 보호 조치 강제화**: 교사 실명 데이터 필터링을 구축하여 플랫폼 내 익명 비방 및 악성 공동 민원 모의를 사전 차단합니다.
* **소상공인 보호 및 법적 분쟁 예방**: 학원의 명예훼손/영업방해 이의제기 시 30일간 비공개 가림 처리하는 '임시조치 창구'를 운영하며 객관적 평가 템플릿 리뷰 방식을 적용합니다.
* **지역 혐오 방지**: 전국 광장 피드 조회 시 닉네임의 특정 법정동 세부 주소를 광역 마스킹하여 동네/아파트별 위화감 조성을 방지합니다.

### 6. 보완이 필요한 리스크 / 맹점 (Blind Spots)
* **초기 데이터 파편화 (Chicken & Egg 문제)**: '우리 학교/학년' 단위로 피드를 격리(UTIL-03)하면, 초기 유저가 적을 때 캘린더 대화방이 텅 비어 보일 수 있습니다. 초기에는 학교 단위를 구/동 단위로 묶어서 보여주다가 유저가 차면 격리하는 '가변형 피드 스코프' 전략이 안전합니다.
* **OCR 어뷰징 구멍 방어**: 영수증의 SHA-256 해시값 중복 체크(RSK-04)는 훌륭하지만, '동일한 영수증을 다른 각도로 찍거나, 모니터 화면을 찍는 행위'는 해시값으로 안 걸립니다. OCR 텍스트 결과물 중 `[학원명 + 승인번호 + 결제일시]`의 조합 중복을 체크하는 로직이 백엔드에 필수적입니다.
* **PDF 마켓의 저작권 이슈**: 선배 맘 노하우 PDF 중 '학교 기출 경향 분석'에 실제 학교 시험지 문항이 그대로 인용될 경우 저작권법 위반 소지가 있습니다. 가이드라인과 필터링 시스템이 필요합니다.

### 7. 개발 단계별 기술 스택 로드맵 (Technology Roadmap)
1. **1단계 (MVP - 시장 검증기 / 1~3개월)**: 최소 비용 및 최단기 출시를 목표로 삼아 대치/반포 지역 학부모 반응을 빠르게 검증합니다.
   - *프론트/앱*: React + Vite + Capacitor (기존 코드 100% 재활용 가능)
   - *백엔드/DB*: **Supabase (BaaS)** - 자체 DB, 구글 Auth, 이미지 저장소 내장 서비스 활용
   - *호스팅/인프라*: Vercel (웹) + Supabase Cloud (서버리스 데이터베이스)
   - *이점*: 백엔드 API 인프라 구축 시간을 80% 이상 단축하고 초기 운영 서버비 거의 0원에 수렴.
2. **2단계 (Scale-up - 서비스 성장기 / 3~6개월)**: 가입자 증가에 따른 독자적 비즈니스 가공 로직 및 데이터 어뷰징 검증을 처리합니다.
   - *프론트/앱*: React (Zustand 전역 상태 관리 및 React Query API 데이터 캐싱 도입) + Capacitor App
   - *백엔드/DB*: **Node.js (Express 또는 NestJS)** 독립 웹 애플리케이션 서버 도입 + **PostgreSQL** + **Redis** (실시간 알림 및 댓글 동기화 캐시) + OpenAI API (비동기 유해 비방 검출 대기열 결합)
   - *호스팅/인프라*: AWS ECS (서버) + RDS (데이터베이스) + S3 (영수증 저장소)
   - *이점*: 중복 영수증 이미지 해시 검증, 악성 학위 저격 차단 필터, 정교한 Q&A 포인트 정산 로직 등 독자적 비즈니스 로직 고도화 가능.
3. **3단계 (Enterprise - 성숙기 / 6개월 이후)**: 대규모 트래픽 수용 및 구글/네이버 SEO 유입을 통한 전국적 스케일로 확장합니다.
   - *프론트/앱*: **Next.js (Web/SEO 유입 및 속도 극대화)** + Capacitor App
   - *백엔드/DB*: **NestJS** (완벽한 구조화 및 마이크로서비스 아키텍처 전환 가능) + PostgreSQL (PostGIS 공간 쿼리 확장으로 위치 기반 편의 서비스 확장)
   - *호스팅/인프라*: AWS ECS Fargate (오토 스케일링 구성) + RDS Multi-AZ (고가용성 이중화) + Redis Cluster + CloudFront (콘텐츠 고속 배포 네트워크)
   - *이점*: 수십만 명의 동시 접속 트래픽 스파이크를 분산 방어하고, 수집된 데이터를 익명 가공해 B2B 사교육 데이터 연구용 API 마켓 구축 가능.

## PART 3. 요즘 트렌드에 맞는 커뮤니티 기능 추천 (Trend-driven Features)

최근 커뮤니티 트렌드는 '긴 글을 읽지 않는 스낵화', 'AI를 통한 정보의 초압축', 그리고 '감정적 연대감의 마이크로화'입니다. Barnet에 적용하여 학부모 충성도와 참여율을 높일 수 있는 트렌디한 기능 4가지를 제안하고 검토합니다.

### 1. AI 라이브 학사/학원 요약 봇 (Information Layer)
* **개요**: 수많은 댓글과 긴 가정통신문, 공지사항을 일일이 읽는 피로감을 AI가 초압축 브리핑으로 해소해 줍니다.
* **기능 구상**: '우리 학교' 캘린더 일정 드로어나 특정 학원 리뷰 탭 상단에 `[AI 3줄 요약]` 버튼을 배치합니다.
* **기능 예시**: *"이번 수행평가는 준비물 유의 필요, 학부모들 사이에서 난이도 상으로 평가됨"* 처럼, 수십 개의 댓글을 AI가 실시간 요약 브리핑해 줍니다.
* **도입 단계**: **2단계 (Scale-up)** 또는 **3단계 (Enterprise)** (OpenAI API 연동 필요)

### 2. 마이크로 오디오 소통방 '등교길 티타임' (Micro-Audio)
* **개요**: 텍스트 형태의 저격이나 박제 우려를 피하고, 동네 엄마들과 가볍게 실시간 수다를 나누고 싶은 니즈를 충족합니다.
* **기능 구상**: 아침 등교 시간(오전 8:30 ~ 9:30)이나 늦은 밤(오후 10시 이후)에 지정된 가명 닉네임 상태로 참여하는 '10분 시간 제한 익명 음성 대화방'을 개설합니다. (예: *"반포초 3학년 오늘 준비물 방"*)
* **기능 예시**: 글을 쓰는 부담 없이 아침 준비물 챙기는 동안 라디오처럼 들으며 소통할 수 있습니다. 음성 대화 특성상 악플이나 비방이 텍스트보다 현저히 적게 발생하는 장점이 있습니다.
* **도입 단계**: **3단계 (Enterprise)** (WebRTC 기반의 실시간 음성 스트리밍 서버 인프라 필요)

### 3. '품앗이' 매칭 및 당근형 교육 마켓 (Hyper-Local Utility)
* **개요**: 커뮤니티 내 정보 교환을 넘어 실질적인 '하이퍼로컬 상호조력 액션'으로 락인(Lock-in) 효과를 냅니다.
* **기능 구상**: 기존 카풀 신청(COMM-05) 기능을 확장하여, `[학원 셔틀 품앗이]`, `[급한 날 아이 픽업 부탁하기]` 등 인증된 회원 간의 상호조력 매칭을 제공합니다. 또한 대치/반포 등 지역 특성상 구하기 어려운 '특정 학원 한정판 교재/프린트 중고 거래 탭'을 신설합니다.
* **기능 예시**: 구글 SSO 및 학교/학원 영수증 인증이 완료된 학부모 간의 거래이므로 신뢰도가 매우 높으며 사기 리스크가 거의 없습니다.
* **도입 단계**: **2단계 (Scale-up)** (1:1 채팅방 개설 및 하이퍼로컬 매칭 알고리즘 필요)

### 4. 익명 밸런스 게임 및 '공감 밈(Meme)' 피드 (Gamification)
* **개요**: 커뮤니티가 지나치게 학습 정보 위주로만 흘러 딱딱해지는 것을 방지하고 매일 앱을 켜게 만드는 게이미피케이션(Gamification) 요소입니다.
* **기능 구상**: 학부모 공감형 익명 밸런스 투표(COMM-02 확장)를 매일 메인 탭에 띄웁니다.
* **기능 예시**: *"이번 주말, 대치동 학원가 라이딩 vs 집에서 홈스쿨링 삼시세끼 챙기기"*, *"담임 선생님 결혼식 부조금 5만 원 vs 10만 원"* 등 가볍고 몰입감 높은 주제로 투표율과 리텐션(재방문율)을 극대화합니다.
* **도입 단계**: **1단계 (MVP)** 또는 **2단계** (기존 커뮤니티 투표 UI 컴포넌트를 확장하여 홈 화면 팝업/배너로 노출 가능)
